### 2.python解释器查询
- **`__doc__` 属性**：
只打印**文档字符串**：每个类或函数定义下方的第一行字符串（用 `"""` 包裹的部分）会被存入该对象的 `__doc__` 属性中
```python
print(str.strip.__doc__) # 获取该函数的文档字符串（docstring）
```
终端：
```bash
Return a copy of the string with leading and trailing whitespace removed.

If chars is given and not None, remove characters in chars instead.
```

- **`help()` 函数**：
包含：
	- **类型描述**：它会告诉你这是一个 `method descriptor`（方法描述符）还是 `class`。
	- **函数签名**：它会自动分析函数的参数（如 `self`, `chars=None`, `/` 等），并显示在第一行。
	- **原始文档**：最后它才会把 `__doc__` 里的内容贴在下面。
```python
help(str.strip) # 直接在控制台打印 strip 方法的详细文档
```
终端：
```bash
Help on method descriptor strip:
strip(self, chars=None, /) unbound builtins.str method
    Return a copy of the string with leading and trailing whitespace removed.

    If chars is given and not None, remove characters in chars instead.
```

#### 1.函数签名
这是每个 API 的最开头，包含了最重要的**默认参数**和**传参规则**。

- **默认值**：如果省略 `chars` 或为 `None`，它会执行默认行为。

- **特殊符号 `/` 和 `*`**：

	- **`/` 之前**：参数只能按位置传，不能写 `key=val`
		
	- **`*` 之后**：参数必须写 `key=val`（强制关键字风格）
#### 2. 隐含行为与数据处理规则

> 文档原文：“Remove leading and trailing characters. The `chars` argument is a string specifying the **set** of characters to be removed.”

这里隐藏了两个关键点：

- **不是“子串”而是“字符集”**：文档特意用了 **set** 这个词。这意味着如果你传入 `"abc"`，它不是寻找整个 `"abc"` 块，而是只要首尾遇到了 `a` 或 `b` 或 `c` 中的**任何一个**，就会剥掉。

- **剥离停止逻辑**：文档隐含了处理规则——它会从两端向中间扫描，直到遇到第一个**不在** `chars` 集合中的字符为止。
